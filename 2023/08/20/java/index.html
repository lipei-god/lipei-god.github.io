3.方法

什么是方法

​ 方法是程序中最小执行单元

什么时候用到方法

重复的代码，具有独立功能的代码可以抽象到方法中

方法的好处

1.提高代码的可执行性

2.提高代码的可维护性

方法定义与格式(三种)

无参方法

有参方法

参数数量跟类型必须一致。（实参形参都要一一对应）

形参：全程形式参数，是指方法定义中的参数
实参: 全程实际参数，方法中调用的参数

带返回值的方法

在调用处根据方法的结果，去编写另外一段代码（结果，在另一个地方有用）

注意

方法不调用就不执行

方法与方法之间是平级关系，不能嵌套定义

方法的编写顺序跟执行顺序无关

方法的返回值类型void，表示方法木有返回值，没有返回值的方法可以省略return语句不写，如果要编写return，后面不能跟具体数值

return语句下面不能写代码，因为永远执行不到，属于无效代码。

return关键字

方法没有返回值：可以省略不写。如果书写，表示结束方法

方法有返回值：必须要写。表示结束方法和返回结果

方法值的传递（2种）

基本数据类型

在栈内存中的方法，每个方法里面的基本数据类型不互相影响。

如果想不同方法之间的基本数据类型有影响就要 传递并且赋值

引用数据类型

对与引用数据类型就不一样了，在栈的不同方法里面修改引用数据类型，由于引用数据类型，指向的是同一个的堆内存。在栈的不同方法里，是可以被修改的

当然，如果方法有返回值，传递并且赋值，也是会修改，但多此一举。

7.面向对象进阶

static

​ static表示静态，是Java中的一个修饰符，可以修饰成员方法，成员变量

​ static 是会在堆空间单独创建一个空间（静态存储对象) 静态变量优先于对象出现

被static修饰的成员变量，叫做静态变量

​ 特点：被该类所有对象共享

​ 调用方法：类名调用（推荐） 或者 对象名调用

被static修饰的成员方法，叫做静态方法

​ 特点：多用在测试类，或者工具类 javabean类很少用

​ 调用方法：类名调用（推荐） 或者 对象名调用

工具类

javabean类 ： 描述一类事物的类

测试类 ： 检查其他类是否正确，带有main的类

工具类：帮助我们做一些事情，但不描述任何事物的类

​ 工具类：

1.类名见名知意

2.私有化构造方法

3.方法定义为静态

static的注意事项

静态方法中，只能访问静态

非静态方法可以访问所有

静态方法中没有this关键字

继承图解

很多类是有公共的属性的，我们可以把公共的属性提取出来，最为父类，然后子类去继承他们的属性

extends关键字

这个关键字，我们可以让一个类和另一个类建立起来继承关系

继承的好处

可以把多个子类中重复的代码抽到父类中去，提高代码的复用性

子类可以在父类的基础上，增加其他功能，是其子类更强大

数据库

数据库的基本操作

1.数据库的创建

语法：create database 数据库名;

数据库只能创建一次。

2.数据库的引用

语法：use 数据库名;

3.查看数据库

语法：show databases;

​ 会查看你现在所存的所有数据库

4.删除数据库

语法：drop database 数据库名;

表的创建语法问题

表创建语法

表的创建的前提是 要有数据库，在数据库已经创建成功的前提下。

语法：

create table 表名
(
列名 数据类型 ,
车辆名 varchar(20),
车牌号 decimal(5,2),
规格 varchar(10),
primary key(列名)
) char set utf8;

注意 , 的使用，在后才加 ; 。

数据类型

**varchar()**：可变长度的字符串，比较智能，节省空间，会根据实际的长度动态分配空间

优点：节省空间

缺点：需要动态分布空间，速度慢

**char()**：定长字符串，不管实际的数据长度是多少，分配固定长度的空间去存储数据

优点：不需要动态分布空间，速度快

缺点：在使用不恰当的时候，可能会导致空间的浪费

关于varchar和char的选择：

性别字段：由于性别字符串的长度是固定的，选择char

姓名字段：每一个人的名字长度不同，选择varchar

int：数字中的整数型,可以不加（），（）里默认为11

bigint：数字中的长整型

float：单精度浮点型数据

double：双精度浮点型数据

decimal（p，d）：精准数据类型，以字符串的形式保存精确的原始数值，而float，double则是非精准类型，在DB中保存的是近似值。p几位数，小数电后面d位。例如decimal（5,2）表示 -999.99~999.99的数。

date：短日期类型

datetime：长日期类型

主键

语法：
primary key（列名）

直接在数据类型后面加 primary key


两种创建，1.直接在列名后面。2.还有在列创建完成后 声明主键

外键

语法：constrait 外键取个名字 foreign key(外键名) referencs 表名（主键名）

自增序列

语法： 只需要在 数据类型后面加 auto_increment

你会发现，序号这一列是我没有插入数据就自动排好数了。

时间

语法：需要在数据类型后面加 default current_timestamp


注意的是 时间的数据类型的 datetime

表的操作

显示表结构

语法: describe 表名;

增加列

语法：alter table 表名 add 列名 数据类型 等限制；

删除列

语法：alter table 表名 drop column 列名；

插入数据

语法：insert into 表名（列1，列2）values(列1对应的数据，列2对应的数据)，（可以插入多个数据）；

修改数据

语法：update 表名 set 原来的数据=更改的数据;

加限制的修改数据

语法： update 表名 set 原来是数据=更改的数据 where 限制语句；


限制地区为null的数据修改

选择（显示）数据

语法：select 列名 from 表名；

​ * 可以代表所有列

加限制的选择（显示）数据

语法：select 列名 from 表名 where 限制条件；

用来连接两个表的条件一般格式

[<表名1>.]<列名1> BETWEEN [<表名2>.]<列名2> AND [<表名2>.]<列名3>

连接条件中的各连接字段类型必须是可比的，但名字不必相同

排序-ORDER BY

可以按照一个或者多个属性列排序

升序：ASC

降序：DESC

缺省值为升序

对于空值

ORDER BY … ASC NULL值会先呈现。

ORDER BY … DESC NULL值会最后呈现

可以先正序排列后，遇到相同的值，根据第二个排序的按倒序排序

一个SELECT-FROM-WHERE语句称为一个查询块，

将一个查询块嵌套在另一个查询块的WHERE子句或者HAVING条件内称为子查询

*子查询不能使用ORDER BY 子句
